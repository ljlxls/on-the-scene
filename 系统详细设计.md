# 系统详细设计

## 1. 系统架构

### 1.1 整体架构
系统采用前后端分离架构，主要由以下组件组成：
- **前端应用**：React + Tailwind CSS + Vite
- **后端服务**：Express.js
- **数据库**：PostgreSQL
- **大模型服务**：集成多种大模型API
- **智能体系统**：基于Hermes设计思路的角色智能体

### 1.2 核心模块
1. **用户模块**：用户注册、登录、个人信息管理
2. **小说模块**：小说上传、分析、管理
3. **角色模块**：角色生成、管理、选择
4. **会话模块**：角色扮演会话管理
5. **智能体模块**：角色智能体的创建、管理、交互
6. **模型配置模块**：大模型API配置管理

## 3. 后端设计

### 3.1 API设计

#### 3.1.1 用户API
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `GET /api/auth/profile` - 获取用户信息
- `PUT /api/auth/profile` - 更新用户信息

#### 3.1.2 小说API
- `POST /api/novels` - 上传小说
- `GET /api/novels` - 获取用户小说列表
- `GET /api/novels/:id` - 获取小说详情
- `GET /api/novels/:id/analysis` - 获取小说分析结果

#### 3.1.3 会话API
- `POST /api/sessions` - 创建角色扮演会话
- `GET /api/sessions` - 获取用户会话列表
- `GET /api/sessions/:id` - 获取会话详情
- `POST /api/sessions/:id/messages` - 发送消息
- `GET /api/sessions/:id/messages` - 获取会话消息

#### 3.1.4 智能体API
- `GET /api/agents/:sessionId` - 获取会话中的智能体
- `PUT /api/agents/:id` - 更新智能体配置

#### 3.1.5 模型配置API
- `GET /api/models` - 获取模型配置列表
- `POST /api/models` - 创建模型配置
- `PUT /api/models/:id` - 更新模型配置
- `DELETE /api/models/:id` - 删除模型配置

### 3.2 服务层设计

#### 3.2.1 用户服务

## 4. 智能体设计

### 4.1 智能体结构

#### 4.1.1 上帝之手智能体
- 负责分析小说
- 生成角色信息
- 描述故事背景和情节
- 协调其他角色智能体的交互

#### 4.1.2 角色智能体

- 拥有独立的灵魂文件、记忆文件和习惯文件

## 5. 数据库设计

### 5.1 表结构

#### 5.1.1 用户表(users)
- id: SERIAL PRIMARY KEY
- email: VARCHAR(255) UNIQUE NOT NULL
- password_hash: VARCHAR(255) NOT NULL
- created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP

#### 5.1.2 小说表(novels)
- id: SERIAL PRIMARY KEY
- user_id: INTEGER REFERENCES users(id)
- title: VARCHAR(255) NOT NULL
- content: TEXT NOT NULL
- status: VARCHAR(50) DEFAULT 'uploaded'
- created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP

#### 5.1.3 角色表(characters)
- id: SERIAL PRIMARY KEY
- novel_id: INTEGER REFERENCES novels(id)
- name: VARCHAR(255) NOT NULL
- description: TEXT
- created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP

#### 5.1.4 会话表(sessions)
- id: SERIAL PRIMARY KEY
- user_id: INTEGER REFERENCES users(id)
- novel_id: INTEGER REFERENCES novels(id)
- character_id: INTEGER REFERENCES characters(id)
- status: VARCHAR(50) DEFAULT 'active'
- created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP

#### 5.1.5 消息表(messages)
- id: SERIAL PRIMARY KEY
- session_id: INTEGER REFERENCES sessions(id)
- content: TEXT NOT NULL
- type: VARCHAR(50) NOT NULL
- sender_id: INTEGER
- created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP

#### 5.1.6 智能体表(agents)
- id: SERIAL PRIMARY KEY
- session_id: INTEGER REFERENCES sessions(id)
- character_id: INTEGER REFERENCES characters(id)
- model_config_id: INTEGER REFERENCES model_configs(id)
- created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP

#### 5.1.7 模型配置表(model_configs)
- id: SERIAL PRIMARY KEY
- user_id: INTEGER REFERENCES users(id)
- name: VARCHAR(255) NOT NULL
- api_key: VARCHAR(255) NOT NULL
- endpoint: VARCHAR(255) NOT NULL
- model_name: VARCHAR(255) NOT NULL
- created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP

#### 5.1.8 灵魂文件表(soul_files)
- id: SERIAL PRIMARY KEY
- agent_id: INTEGER REFERENCES agents(id)
- content: TEXT NOT NULL
- created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP

#### 5.1.9 记忆文件表(memory_files)
- id: SERIAL PRIMARY KEY
- agent_id: INTEGER REFERENCES agents(id)
- content: TEXT NOT NULL
- created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP
- updated_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP

#### 5.1.10 习惯文件表(habit_files)
- id: SERIAL PRIMARY KEY
- agent_id: INTEGER REFERENCES agents(id)
- content: TEXT NOT NULL
- created_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP
- updated_at: TIMESTAMP DEFAULT CURRENT_TIMESTAMP

## 6. 系统流程

### 6.1 小说上传与分析流程
1. 用户上传小说文件或输入小说文本
2. 系统存储小说内容
3. 调用"上帝之手"大模型分析小说
4. 大模型提取小说中的主要角色（3-5个）
5. 生成每个角色的基本信息
6. 展示角色列表供用户选择

### 6.2 角色扮演流程
1. 用户选择一个角色进行扮演
2. 系统为每个角色创建智能体
3. 为每个智能体生成灵魂文件、记忆文件和习惯文件
4. "上帝之手"智能体描述故事背景
5. 各角色智能体根据灵魂文件和情节发展互动
6. 用户输入指令控制扮演的角色
7. 系统处理用户输入并更新会话状态
8. 智能体更新记忆文件和习惯文件
9. "上帝之手"智能体推进故事发展
10. 重复步骤5-9直到会话结束

### 6.3 智能体交互流程
1. "上帝之手"智能体生成故事场景描述
2. 系统将场景描述发送给所有角色智能体
3. 每个角色智能体根据自身的灵魂文件、记忆文件和习惯文件生成响应
4. 系统收集所有角色智能体的响应
5. "上帝之手"智能体整合所有响应，推进故事发展
6. 重复步骤1-5直到会话结束

## 7. 技术实现要点

### 7.1 前端实现
- 使用React 18的新特性，如Concurrent Mode和Suspense
- 使用Tailwind CSS实现响应式设计
- 使用React Router实现页面路由
- 使用Axios处理API请求
- 实现WebSocket连接，支持实时会话

### 7.2 后端实现
- 使用Express.js构建RESTful API
- 使用PostgreSQL存储数据
- 使用Sequelize作为ORM
- 使用JWT进行身份验证
- 实现中间件处理跨域请求和错误

### 7.3 智能体实现
- 基于Hermes设计思路，实现三层能力架构
- 使用文件系统存储智能体的灵魂文件、记忆文件和习惯文件
- 实现智能体的自进化能力
- 支持多种大模型API的集成

### 7.4 大模型集成
- 支持OpenAI、Anthropic等多种大模型API
- 实现模型调用的抽象层，便于切换不同模型
- 处理模型的速率限制和错误重试
- 优化模型调用的成本和性能
